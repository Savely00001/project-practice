# 1. Техническое руководство 

Это руководство проведёт вас через создание 2D RPG шаг за шагом. Предполагается, 
что вы уже знакомы с основами Python и установили Pygame (pip install pygame в cmd от имени администратора).

## 1.1. Шаг 1: Создание игрового окна и цикла

```Python
import pygame
import sys

pygame.init()
SCREEN_WIDTH = 1200
SCREEN_HEIGHT = 600
screen = pygame.display.set_mode((SCREEN_WIDTH, SCREEN_HEIGHT))
pygame.display.set_caption("RPG Game")
clock = pygame.time.Clock()
FPS = 60

running = True
while running:
    for event in pygame.event.get():
        if event.type == pygame.QUIT:
            running = False

    screen.fill((30, 30, 50))
    pygame.display.flip()
    clock.tick(FPS)

pygame.quit()
sys.exit()
```
## 1.2. Шаг 2: Класс игрока (движение и анимация)
Создайте класс Player с атрибутами x, y, speed, направлением, флагом is_moving. 
Для анимации загрузите спрайт листы (8 кадров в строке, размер кадра 96×80). 
Масштабируйте по желанию.

Ключевые моменты:
1.	Метод __init__ загружает спрайт-листы и нарезает кадры.
2.	Метод update_animation() меняет кадр на основе прошедшего времени.
3.	Метод move() обновляет позицию и направление, сбрасывает кадр при смене направления.

Пример загрузки анимации для одного направления:
```Python
sheet = pygame.image.load(path).convert_alpha()
frames = []
for i in range(8):
    rect = pygame.Rect(i * 96, 0, 96, 80)
    frame = sheet.subsurface(rect)
    frames.append(pygame.transform.scale(frame, (self.width, self.height)))
self.run_anim[direction] = frames

Отрисовка в главном файле:
player.move(dx, dy)   # dx, dy вычисляются из нажатых клавиш
```

## 1.3. Шаг 3: Классы NPC, предметы и квесты
1.	NPC – хранит имя, спрайт, флаг врага, диалоги, характеристики (для врагов).
2.	Item – название, тип (potion, weapon, armor, gold), цена, эффект. Метод use(player) возвращает True, если предмет расходуется.
3.	Quest – цели (описание, целевое количество, текущее количество), награды. Метод update_objective() увеличивает прогресс и проверяет завершение.

Пример предмета:
```Python
class Item:
    def __init__(self, name, desc, item_type, value, effect, icon_path=None):
        self.name = name
        self.type = item_type
        self.value = value
        self.effect = effect
        self.icon_path = icon_path
```

## 1.5. Шаг 5: Интерфейс инвентаря
Функция show_inventory() рисует окно с экипировкой и сеткой предметов. 
Подсказки при наведении мыши, снятие экипировки по R. Используются методы equip_item и unequip класса Player.

Основные элементы инвентаря:

1.	Слоты оружия и брони
2.	Сетка предметов (5 столбцов)
3.	При наведении на предмет – всплывающая подсказка с названием, типом, эффектом, ценой
4.	При наведении на слот экипировки и нажатии R – предмет снимается и возвращается в инвентарь

## 1.6. Шаг 6: Боевая система
Функция start_combat создаёт боевой экран. 
Она содержит собственный цикл, который выполняется до победы или побега.

Отрисовка:

1.	Карточка врага слева вверху (спрайт 120×120, полоска HP)
2.	Карточка игрока справа внизу
3.	Меню действий (4 кнопки) – АТАКА, РЮКЗАК, СТАТУС, УБЕЖАТЬ
4.	Диалоговое окно вывода сообщений

Управление: A/D – выбор действия, SPACE – подтверждение.
Логика атаки:
```Python
damage = max(1, player.attack - enemy.defense)
enemy.hp -= damage
dialogue_text = f"Вы нанесли {damage} урона!"
if enemy.hp <= 0:
    # награда, обновление квеста
    return True
# ответ врага
enemy_damage = max(1, enemy.attack - player.defense)
player.hp -= enemy_damage
dialogue_text += f" Враг атакует! -{enemy_damage} HP"
```
РЮКЗАК вызывает show_battle_inventory – окно только с расходуемыми предметами (зельями).

СТАТУС показывает параметры врага в диалоговом окне.

УБЕЖАТЬ восстанавливает здоровье врага и завершает бой.

# 2. Модификация проекта: система сохранения и загрузки
## 2.1. Цель модификации
Оригинальная игра не сохраняла прогресс. 
При перезапуске всё начиналось заново. Добавлена возможность сохранять и загружать состояние игры.

## 2.2. Выбор формата
Использован модуль json – данные хранятся в файле savegame.json в удобочитаемом виде.

## 2.3. Сохраняемые данные
1.	Координаты игрока (player.x, player.y)
2.	Уровень, здоровье, максимальное здоровье
3.	Базовая атака и защита
4.	Количество золота
5.	Инвентарь (список предметов в виде словарей)
6.	Экипированное оружие и броня
7.	Прогресс по квесту
8.	Состояние монеты на карте (coin_taken)
9.	Содержимое сундука (chest_items)
10.	Состояние врагов (HP, позиции)

## 2.4. Реализация
Добавлены функции save_game() и load_game(). 
Пример сохранения инвентаря:
```Python 
'inventory': [{'name': i.name, 'type': i.type, 'effect': i.effect, 'value': i.value} for i in player.inventory]
```
При загрузке словари преобразуются обратно в объекты Item (требуется маппинг названий на шаблоны или восстановление по атрибутам).
Вызов сохранения и загрузки добавлен в основной цикл:

1.	Клавиша F5 – сохранить игру
2.	Клавиша F9 – загрузить последнее сохранение
После загрузки игрок появляется в сохранённой позиции, восстанавливаются все предметы, золото, здоровье и прогресс квеста.

## 2.5. Результат модификации
Игра стала полноценной – игрок может прервать сессию и позже продолжить с того же места. 
Это значительно улучшает пользовательский опыт.
